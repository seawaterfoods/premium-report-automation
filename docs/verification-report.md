## 三、結構格式差異清單（需調整項目）

### 差異 1：表頭險種名稱缺少換行符

| 影響頁簽 | 115單, 115單累 |
|----------|---------------|
| **嚴重度** | 低 (視覺排版) |
| **說明** | 範例中險種短名有 `\n` 換行（如 `"一年期\n住宅火險"`），程式輸出為單行 `"一年期住宅火險"` |
| **原因** | `CategoryMapping.CODE_TO_SHORT_NAME` 未加入換行符 |
| **修復** | 在短名定義中加入 `\n` |

### 差異 2：險種代號儲存型態

| 影響頁簽 | 115單 Row 1, 115單累 Row 1 |
|----------|--------------------------|
| **嚴重度** | 低 (功能不影響) |
| **說明** | 範例中 2000~9900 代號為數字型態，程式統一為文字型態 |
| **原因** | 規格書規定「險種代號統一轉為文字」 |
| **修復** | **不需修復** — 這是規格要求。若需一致，可改為數字輸出 |

### 差異 3：115總/115總累 表頭結構

| 影響頁簽 | 115總, 115總累 |
|----------|---------------|
| **嚴重度** | 中 (欄位位置偏移) |
| **說明** | |

**範例表頭結構 (22 欄 A~V)：**
```
Row 1: [A-V 合併] "115年度 3 月份各保險公司簽單保費統計總表"
Row 2: [V] "單位:新台幣元"
Row 3: [G合併G:K] "汽車險" | [L合併L:O] "意外險" | [V] "比較"
```

## 六、結論

### ✅ 正面結果
1. **核心 ETL 引擎完全正確** — 33/33 險種數值與來源完全一致
2. **比人工更準確** — 範例有 31 處人工輸入錯誤，程式輸出為 0 處
3. **合計計算正確** — 來源 1,035,357,275 = 程式 1,035,357,275
4. **8 個頁簽全部產生** — 結構邏輯正確
5. **成長率邊界處理正確** — 缺去年資料時以 1 為分母並記錄警告

### ✅ 已修復 (commit 6f62666)
1. **P1** 115總/總累 — 3層合併表頭 (22欄 A-V，含國外分進)
2. **P2** 比較增減率 — 3層合併表頭 (汽車險/意外險/強制責任分組)
3. **P3** 險種短名加 `\n` 換行符 (8個短名)
4. **P4** 歸屬改為 5 欄結構 (類/歸屬/代號/子分組/險種全名)
5. **P5** 增減原因標題+結構對齊範例
6. **P6** 排名簡化結構 (無標題行、無排名欄)

### ⚠️ 待後續處理
1. **需用完整資料再測** — 目前僅 1 家公司測試，需用 19 家完整資料驗證
