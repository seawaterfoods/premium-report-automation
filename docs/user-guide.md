## 1. 系統概述

將多家保險公司的「保險收入統計表」Excel 來源檔案，自動彙整產出：

| 輸出報表 | 頁簽 | 說明 |
|---------|------|------|
| `{YYY}年產險業務(簽單)保費統計表.xlsx` | 5 個 | 明細(單月/累計)、總表(單月/累計)、歸屬對照 |
| `{YYMM}vs{YYMM}同期比較分析表.xlsx` | 2 個 | 比較增減率、增減原因 |

## 3. 資料夾結構

```
premium-report-automation/
├── config/
│   └── application.yml          ← 主要設定檔 (修改這個)
├── import/                      ← 來源資料夾
│   ├── 115/                     ← 今年 (民國年)
│   │   ├── 01/                  ← 月份子資料夾
│   │   │   ├── 01_11501_保險收入統計表.xlsx
│   │   │   ├── 02_11501_保險收入統計表.xlsx
│   │   │   └── ...
│   │   ├── 02/
│   │   └── 03/
│   └── 114/                     ← 去年 (自動掃描)
│       ├── 01/
│       ├── 02/
│       └── 03/
├── output/                      ← 輸出資料夾
│   ├── 11503/                   ← 按年月分資料夾
│   │   ├── 115年產險業務(簽單)保費統計表.xlsx
│   │   └── 11503vs11403同期比較分析表.xlsx
│   └── report.log               ← 執行紀錄 (每次覆蓋)
├── build.bat                    ← 編譯腳本
├── run.bat                      ← 執行腳本
└── src/                         ← 原始碼
```

### 來源檔案命名規則

```
{公司代號}_{民國年月4碼}_保險收入統計表.xlsx
```

| 欄位 | 說明 | 範例 |
|------|------|------|
| 公司代號 | 1~2 位數字 | `01`, `12`, `5` |
| 民國年月 | 3 位年 + 2 位月 | `11503` = 115年3月 |

範例檔名：
- `12_11503_保險收入統計表.xlsx` — 公司 12（旺旺友聯）的 115 年 3 月資料
- `01_11501_保險收入統計表.xlsx` — 公司 01（台產）的 115 年 1 月資料

### 月份資料夾

月份子資料夾名稱可用 `01`~`12` 或 `1`~`12`（兩種皆支援）。

## 4. 設定檔說明

設定檔位於 `config/application.yml`，以下為完整範例及說明：

```yaml
app:
  # ===== 路徑設定 =====
  import-dir: ./import          # 來源資料夾路徑
  output-dir: ./output          # 輸出資料夾路徑

  # ===== 年份設定 =====
  process-year: 115             # 處理哪一年 (民國年)
                                # 系統會自動掃描去年 (114) 的資料夾

  # ===== 險種欄位控制 =====
  columns:
    hidden-codes:               # 隱藏的險種代號清單
      - "9900"                  # 隱藏國外分進
      # - "3300"               # 未來新增的險種也可以在這裡隱藏
    # 說明:
    # - 明細表 (115單/115單累): 隱藏指定代號的欄位
    # - 總表 T 欄合計: 若 "9900" 在清單中，SUM 公式會排除國外分進
    # - 設為空清單 [] 則顯示全部欄位

  # ===== 公司排序 =====
  company-order: BY_CODE_ASC    # 選項: BY_CODE_ASC | BY_CODE_DESC | BY_NAME
```

### 設定項目速查表

| 設定 | 值 | 說明 |
|------|-----|------|
| `import-dir` | 路徑 | 來源資料夾，含 `{年份}/{月份}/` 子結構 |
| `output-dir` | 路徑 | 輸出根目錄，系統會自動建立 `{年月}/` 子資料夾 |
| `process-year` | 數字 | 民國年，如 `115`。系統自動掃描 `114` 作為去年 |
| `hidden-codes` | 清單 | 要隱藏的險種代號，如 `["9900"]` 或 `["9900", "3300"]` |
| `company-order` | 列舉 | 公司排序方式 |

## 5. 操作流程

### Step 1: 放置來源檔案

```
import/
├── 115/
│   └── 03/
│       ├── 01_11503_保險收入統計表.xlsx
│       ├── 02_11503_保險收入統計表.xlsx
│       ├── 05_11503_保險收入統計表.xlsx
│       └── ...
└── 114/
    └── 03/
        ├── 01_11403_保險收入統計表.xlsx
        ├── 02_11403_保險收入統計表.xlsx
        └── ...
```

> ⚠️ **重要**：去年資料要放在 `import/{去年}/` 資料夾中，否則 U 欄（去年合計）會顯示為 0。

### Step 2: 確認設定

編輯 `config/application.yml`：
- 確認 `process-year` 是否正確
- 確認 `hidden-codes` 是否需要調整

### Step 3: 首次使用先編譯

```cmd
build.bat
```

或手動：
```cmd
mvnw.cmd clean package -DskipTests
```

### Step 4: 執行

```cmd
run.bat
```

或手動：
```cmd
java -jar target/premium-report-automation-0.0.1-SNAPSHOT.jar
```

### Step 5: 確認輸出

輸出會在 `output/{年月}/` 資料夾中，例如 `output/11503/`。

同時檢查 `output/report.log` 確認是否有錯誤訊息。

## 6. 輸出說明

### 報表一：保費統計表

| 頁簽 | 說明 |
|------|------|
| `{YYY}單` | 各公司各月份的 33 險種明細 (單月) |
| `{YYY}單累` | 各公司各月份的 33 險種明細 (累計) |
| `{YYY}總` | 九大類子分類彙總 + 去年合計 + 成長率 (單月) |
| `{YYY}總累` | 九大類子分類彙總 + 去年合計 + 成長率 (累計) |
| 歸屬 | 險種代號 → 九大類的歸屬對照表 |

### 報表二：同期比較分析表

| 頁簽 | 說明 |
|------|------|
| 比較增減率 | 當月 & 累計的今年/去年比較 (差異、佔比、成長率) |
| 增減原因 | 各險種差異金額與佔比 (原因欄位留空，需手動填寫) |

### 欄位說明（總表）

| 欄位 | 說明 |
|------|------|
| D~S | 16 個子分類金額 (火險、水險、航空…國外分進) |
| T | 今年合計 = SUM(D:R) 或 SUM(D:S)，取決於 hidden-codes 設定 |
| U | 去年同期合計 (從去年來源檔計算) |
| V | 成長率 = (T-U)/U (去年為 0 或負數時顯示 #N/A) |

## 7. 新增險種教學

當保險業新增一個險種（例如代號 `3300`），需要修改以下檔案：

### 7.1 修改 `CategoryMapping.java`

路徑：`src/main/java/com/insurance/report/model/CategoryMapping.java`

#### (a) 加入代號清單

在 `ALL_INSURANCE_CODES` 清單中新增代號（依序插入適當位置）：

```java
public static final List<String> ALL_INSURANCE_CODES = List.of(
        // ... 既有代號 ...
        "3000", "3100", "3200",
        "3300",    // ← 新增
        "9900"
);
```

#### (b) 加入短名 (表頭用)

在 `CODE_TO_SHORT_NAME` 中新增：

```java
map.put("3300", "新險種\n短名");   // ← 新增 (可用 \n 換行)
```

#### (c) 加入全名 (歸屬表用)

在 `CODE_TO_FULL_NAME` 中新增：

```java
map.put("3300", "新險種全名");     // ← 新增
```

#### (d) 歸屬到子分類

在 `SUB_CATEGORIES` 的 static block 中，將 3300 加入適當的子分類：

**情況 A — 歸屬到既有子分類：**
```java
// 例如歸屬到健康險
list.add(new SubCategory("健康險", "健康險", List.of("3000", "3100", "3300")));
//                                                                    ↑ 加入
```

**情況 B — 建立新子分類：**
```java
list.add(new SubCategory("新子分類名", "歸屬大類名", List.of("3300")));
```

> ⚠️ 如果建立新子分類，總表會自動多一欄，但表頭合併儲存格可能需要調整
> `PremiumReportWriter.writeSummarySheet()` 中的 group/merge 邏輯。

#### (e) 歸屬表子分組 (可選)

如果需要在歸屬表的 D 欄顯示子分組名稱，在 `CODE_TO_SUB_GROUP` 新增：

```java
map.put("3300", "新分組");
```

### 7.2 修改來源解析器 (通常不需要)

`InsuranceCodeUtil.java` 會自動辨識 4 位數代號。如果新險種的代號格式不同，才需要修改。

來源 Excel 中的欄位是按固定 row 位置解析的（`SourceExcelReader.java`），如果來源檔案結構改變，需要同步調整。

### 7.3 暫時隱藏新險種

如果新險種的來源資料尚未就緒，可以在 `config/application.yml` 中暫時隱藏：

```yaml
columns:
  hidden-codes:
    - "9900"
    - "3300"    # 暫時隱藏新險種
```

### 7.4 修改完整範例

假設新增 `3300` 歸屬到「健康險」大類下的新子分類「長照險」：

```java
// 1. ALL_INSURANCE_CODES — 在 3200 之後插入
"3000", "3100", "3200", "3300", "9900"

// 2. CODE_TO_SHORT_NAME
map.put("3300", "長照險");

// 3. CODE_TO_FULL_NAME
map.put("3300", "長期照顧保險");

// 4. SUB_CATEGORIES — 新增子分類
list.add(new SubCategory("長照險", "健康險", List.of("3300")));

// 5. 不需要修改 hidden-codes (除非要暫時隱藏)
```

### 7.5 重新編譯

修改程式碼後需要重新編譯：

```cmd
build.bat
```

## 8. 常見問題

### Q1: 執行後顯示「找不到去年(114)資料，去年數值將預設為 0」

**原因**：`import/114/` 資料夾不存在或裡面沒有符合格式的檔案。

**解法**：
1. 建立 `import/114/{月份}/` 資料夾
2. 放入去年的保險收入統計表 (檔名格式須正確)

### Q2: 明細表的某個險種欄位消失了

**原因**：該代號被加入 `config/application.yml` 的 `hidden-codes` 清單。

**解法**：從 `hidden-codes` 中移除該代號。

### Q3: 總表 T 欄合計數字不對

**原因**：可能 `hidden-codes` 包含 `9900`，導致 T 欄 SUM 公式排除國外分進。

**解法**：
- 若要包含國外分進：從 `hidden-codes` 移除 `"9900"`
- 若要排除：保持 `"9900"` 在清單中

### Q4: 成長率顯示 #N/A

**原因**：去年數值為 0 或負數，系統自動以 1 為分母計算並顯示 #N/A。

**說明**：詳細記錄可查看 `output/report.log` 中的 WARN 訊息。

### Q5: 檔名格式不符被跳過

**原因**：來源檔案名稱不符合 `{公司代號}_{年月}_保險收入統計表.xlsx` 格式。

**解法**：確認檔名格式，公司代號為 1~2 位數字，年月為 5 位數字 (如 11503)。

### Q6: 多次執行輸出互相覆蓋？

**說明**：同月份的報表會輸出到相同的 `output/{年月}/` 資料夾，重複執行會覆蓋。不同月份的報表會在各自的子資料夾中。
