## 1. 系統概述

將多家保險公司的「保險收入統計表」Excel 來源檔案，自動彙整產出：

| 輸出報表 | 頁簽 | 說明 |
|---------|------|------|
| `{YYY}年產險業務(簽單)保費統計表.xlsx` | 5 個 | 明細(單月/累計)、總表(單月/累計)、歸屬對照 |
| `{YYMM}vs{YYMM}同期比較分析表.xlsx` | 2 個 | 比較增減率、增減原因 |

```
│   └── insurance-mapping.yml    ← 險種歸屬設定 (代號、名稱、分類)
├── import/                      ← 來源資料夾
│   ├── 115/                     ← 今年 (民國年)
│   │   ├── 01/                  ← 月份子資料夾
│   │   │   ├── 01_11501_保險收入統計表.xlsx
│   │   │   ├── 02_11501_保險收入統計表.xlsx
│   │   │   └── ...
│   │   ├── 02/
│   │   └── 03/
│   └── 114/                     ← 去年 (自動掃描)
│       ├── 01/
│       ├── 02/
│       └── 03/
├── output/                      ← 輸出資料夾
│   ├── 11503/                   ← 按年月分資料夾
│   │   ├── 115年產險業務(簽單)保費統計表.xlsx
│   │   └── 11503vs11403同期比較分析表.xlsx

├── build.bat                    ← 編譯腳本
├── run.bat                      ← 執行腳本
└── src/                         ← 原始碼
```

### 來源檔案命名規則

```
{公司代號}_{民國年月4碼}_保險收入統計表.xlsx
```

| 欄位 | 說明 | 範例 |
|------|------|------|
| 公司代號 | 1~2 位數字 | `01`, `12`, `5` |
| 民國年月 | 3 位年 + 2 位月 | `11503` = 115年3月 |

範例檔名：
- `12_11503_保險收入統計表.xlsx` — 公司 12（旺旺友聯）的 115 年 3 月資料
- `01_11501_保險收入統計表.xlsx` — 公司 01（台產）的 115 年 1 月資料

### 月份資料夾

月份子資料夾名稱可用 `01`~`12` 或 `1`~`12`（兩種皆支援）。

## 4. 設定檔說明

設定檔位於 `config/application.yml`，以下為完整範例及說明：

```yaml
app:
  # ===== 路徑設定 =====
  import-dir: ./import          # 來源資料夾路徑
  output-dir: ./output          # 輸出資料夾路徑

  # ===== 年份設定 =====
  process-year: 115             # 處理哪一年 (民國年)
                                # 系統會自動掃描去年 (114) 的資料夾

  # ===== 險種欄位控制 =====
  columns:
    hidden-codes:               # 隱藏的險種代號清單
      - "9900"                  # 隱藏國外分進
      # - "3300"               # 未來新增的險種也可以在這裡隱藏
    # 說明:
    # - 明細表 (115單/115單累): 隱藏指定代號的欄位
    # - 總表 T 欄合計: 若 "9900" 在清單中，SUM 公式會排除國外分進
    # - 設為空清單 [] 則顯示全部欄位

  # ===== 公司排序 =====
  company-order: BY_CODE_ASC    # 選項: BY_CODE_ASC | BY_CODE_DESC | BY_NAME
```

### 設定項目速查表

| 設定 | 值 | 說明 |
|------|-----|------|
| `import-dir` | 路徑 | 來源資料夾，含 `{年份}/{月份}/` 子結構 |
| `output-dir` | 路徑 | 輸出根目錄，系統會自動建立 `{年月}/` 子資料夾 |
| `process-year` | 數字 | 民國年，如 `115`。系統自動掃描 `114` 作為去年 |
| `hidden-codes` | 清單 | 要隱藏的險種代號，如 `["9900"]` 或 `["9900", "3300"]` |
| `company-order` | 列舉 | 公司排序方式 |

### 險種歸屬設定檔

設定檔位於 `config/insurance-mapping.yml`，定義險種代號、名稱、歸屬分類。
**修改此檔案不需要重新編譯程式。**

結構概覽：
```yaml
app:
  insurance:
    codes:          # 險種代號清單 (順序=明細表欄位順序)
      - code: "0100"
        short-name: "一年期\n住宅火險"   # 表頭顯示名
        full-name: "一年期住宅火災保險"  # 歸屬表顯示名
    categories:     # 九大類歸屬
      - name: "火險"
        number: "一"
        sub-categories:
          - name: "火險"
            codes: ["0100", "0200", "0300", "0400"]
```

子分類可選欄位：
| 欄位 | 說明 |
|------|------|
| `sub-group` | 歸屬表 D 欄的子分組標籤 |
| `header-group` | 總表/比較表中多個子分類共享的合併標題 |
| `header-label` | 合併標題下個別欄位的名稱 |
| `short-header` | 兩行顯示的第一行文字 |
| `sub-header` | 兩行顯示的第二行文字 |

## 5. 操作流程

### Step 1: 放置來源檔案

```
import/
├── 115/
│   └── 03/
│       ├── 01_11503_保險收入統計表.xlsx
│       ├── 02_11503_保險收入統計表.xlsx
│       ├── 05_11503_保險收入統計表.xlsx
│       └── ...
└── 114/
    └── 03/
        ├── 01_11403_保險收入統計表.xlsx
        ├── 02_11403_保險收入統計表.xlsx
        └── ...
```

> ⚠️ **重要**：去年資料要放在 `import/{去年}/` 資料夾中，否則 U 欄（去年合計）會顯示為 0。

### Step 2: 確認設定

編輯 `config/application.yml`：
- 確認 `process-year` 是否正確
- 確認 `hidden-codes` 是否需要調整

### Step 3: 首次使用先編譯

```cmd
build.bat
```

或手動：
```cmd
mvnw.cmd clean package -DskipTests
```

### Step 4: 執行

```cmd
run.bat
```

或手動：
```cmd
java -jar target/premium-report-automation-0.0.1-SNAPSHOT.jar
```

### Step 5: 確認輸出

輸出會在 `output/{年月}/` 資料夾中，例如 `output/11503/`。

同時檢查 `output/report.log` 確認是否有錯誤訊息。

## 6. 輸出說明

### 報表一：保費統計表

| 頁簽 | 說明 |
|------|------|
| `{YYY}單` | 各公司各月份的 33 險種明細 (單月) |
| `{YYY}單累` | 各公司各月份的 33 險種明細 (累計) |
| `{YYY}總` | 九大類子分類彙總 + 去年合計 + 成長率 (單月) |
| `{YYY}總累` | 九大類子分類彙總 + 去年合計 + 成長率 (累計) |
| 歸屬 | 險種代號 → 九大類的歸屬對照表 |

### 報表二：同期比較分析表

| 頁簽 | 說明 |
|------|------|
| 比較增減率 | 當月 & 累計的今年/去年比較 (差異、佔比、成長率) |
| 增減原因 | 各險種差異金額與佔比 (原因欄位留空，需手動填寫) |

### 欄位說明（總表）

| 欄位 | 說明 |
|------|------|
| D~S | 16 個子分類金額 (火險、水險、航空…國外分進) |
| T | 今年合計 = SUM(D:R) 或 SUM(D:S)，取決於 hidden-codes 設定 |
| U | 去年同期合計 (從去年來源檔計算) |
| V | 成長率 = (T-U)/U (去年為 0 或負數時顯示 #N/A) |

## 7. 新增險種教學

當保險業新增一個險種（例如代號 `3300` 長照險），**只需修改 YAML 設定檔，不需要改程式碼或重新編譯**。

### 7.1 修改 `config/insurance-mapping.yml`

#### (a) 新增代號定義

在 `codes` 區塊中，於適當位置新增一筆（順序 = 明細表欄位順序）：

```yaml
codes:
  # ... 既有代號 ...
  - code: "3100"
    short-name: "長年期\n健康險"
    full-name: "長年期健康險"
  - code: "3300"           # ← 新增
    short-name: "長照險"    # ← 明細表表頭顯示名 (可用 \n 換行)
    full-name: "長期照顧保險" # ← 歸屬表顯示名
  - code: "3200"
    short-name: "強制微型電動\n二輪車責險"
    full-name: "強制微型電動二輪車責任險"
```

#### (b) 歸屬到子分類

在 `categories` 區塊中，找到對應的大類，將代號加入 `codes` 清單：

**情況 A — 歸屬到既有子分類（例如健康險）：**
```yaml
- name: "健康險"
  number: "八"
  sub-categories:
    - name: "健康險"
      codes: ["3000", "3100", "3300"]  # ← 加入 3300
```

**情況 B — 建立新子分類：**
```yaml
- name: "健康險"
  number: "八"
  sub-categories:
    - name: "健康險"
      codes: ["3000", "3100"]
    - name: "長照險"           # ← 新子分類
      codes: ["3300"]
```

> 新增子分類會自動在總表/比較表新增對應欄位，表頭也會自動調整。

#### (c) 歸屬表子分組 (可選)

如果需要在歸屬表的 D 欄顯示子分組名稱，加入 `sub-group`：

```yaml
- name: "長照險"
  sub-group: "長照"        # ← 歸屬表 D 欄顯示
  codes: ["3300"]
```

#### (d) 總表/比較表子分組標題 (可選)

如果新子分類需要和其他子分類共享一個合併標題（類似「強制責任險」包含汽車/機車/電動二輪），
使用 `header-group` 和 `header-label`：

```yaml
- name: "一般長照"
  header-group: "長照險"    # ← 合併標題名
  header-label: "一般"      # ← 個別欄位名
  codes: ["3300"]
- name: "機構長照"
  header-group: "長照險"
  header-label: "機構"
  codes: ["3400"]
```

### 7.2 暫時隱藏新險種

如果新險種的來源資料尚未就緒，在 `config/application.yml` 中隱藏：

```yaml
columns:
  hidden-codes:
    - "9900"
    - "3300"    # 暫時隱藏新險種
```

### 7.3 修改後直接執行

```cmd
run.bat
```

**不需要 `build.bat`，不需要重新編譯。**

### 7.4 完整範例

新增 `3300` 歸屬到「健康險」大類下的新子分類「長照險」：

```yaml
# config/insurance-mapping.yml 的修改:

# 1. codes 區塊 — 在 3100 之後插入
- code: "3300"
  short-name: "長照險"
  full-name: "長期照顧保險"

# 2. categories 區塊 — 健康險大類新增子分類
- name: "健康險"
  number: "八"
  sub-categories:
    - name: "健康險"
      codes: ["3000", "3100"]
    - name: "長照險"
      codes: ["3300"]
```

## 8. 常見問題

### Q1: 執行後顯示「找不到去年(114)資料，去年數值將預設為 0」

**原因**：`import/114/` 資料夾不存在或裡面沒有符合格式的檔案。

**解法**：
1. 建立 `import/114/{月份}/` 資料夾
2. 放入去年的保險收入統計表 (檔名格式須正確)

### Q2: 明細表的某個險種欄位消失了

**原因**：該代號被加入 `config/application.yml` 的 `hidden-codes` 清單。

**解法**：從 `hidden-codes` 中移除該代號。

### Q3: 總表 T 欄合計數字不對

**原因**：可能 `hidden-codes` 包含 `9900`，導致 T 欄 SUM 公式排除國外分進。

**解法**：
- 若要包含國外分進：從 `hidden-codes` 移除 `"9900"`
- 若要排除：保持 `"9900"` 在清單中

### Q4: 成長率顯示 #N/A

**原因**：去年數值為 0 或負數，系統自動以 1 為分母計算並顯示 #N/A。

**說明**：詳細記錄可查看 `output/report.log` 中的 WARN 訊息。

### Q5: 檔名格式不符被跳過

**原因**：來源檔案名稱不符合 `{公司代號}_{年月}_保險收入統計表.xlsx` 格式。

**解法**：確認檔名格式，公司代號為 1~2 位數字，年月為 5 位數字 (如 11503)。

### Q6: 多次執行輸出互相覆蓋？

**說明**：同月份的報表會輸出到相同的 `output/{年月}/` 資料夾，重複執行會覆蓋。不同月份的報表會在各自的子資料夾中。
