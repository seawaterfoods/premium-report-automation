> **版本：** v1.0
> **日期：** 2026-04-17
> **狀態：** 全部需求確認完畢，可進入開發（變更紀錄見附錄）

---

## 一、專案概述

### 1.1 目標

將目前以人工手動操作的「產險業務保費統計表」製作流程自動化。
透過程式讀取各保險公司繳交的保險收入統計表（Excel），
自動彙整、分類加總、產生同期比較分析，輸出為指定格式的 Excel 報表。

### 1.2 背景

- 目前由承辦人員手動將各公司資料逐一輸入 Excel 模板
- 容易發生人工輸入錯誤（已知案例：旺旺友聯 0100 險種數值不一致）
- 每月重複作業，耗時且無附加價值

### 1.3 預期效益

- 消除人工輸入錯誤
- 每月執行只需雙擊 `.bat` 即可完成
- 新增險種時可透過歸屬對照表自動擴展
- 可設定欄位顯示/隱藏，彈性應對需求變化

## 二、技術規格

| 項目 | 規格 |
|------|------|
| 程式語言 | Java 17 |
| 框架 | Spring Boot（支援 Java 17 之最高穩定版本）|
| Excel 處理 | Apache POI（支援 .xlsx）|
| 輸出格式 | **.xlsx**（全面採用 Excel 2007+ 格式）|
| 執行方式 | 雙擊 `.bat` 檔（內部執行 `java -jar xxx.jar`）|
| 設定檔 | `application.yml` 或 `application.properties` |
| 建置工具 | Maven 或 Gradle |
| 計算方式 | **全部由程式直接計算**，不使用 Excel 公式 |
| 數值精度 | 金額為整數（新台幣元）；百分比四捨五入至小數第 2 位 |

### 3.1 資料夾規則

- **依年份建立**頂層資料夾（如 `import/115/`）
- 每個年份下依**月份**建立子資料夾（`01/` ~ `12/`）
- 同月份資料夾內放該月所有公司的來源檔
- **若需去年同期資料**，將去年檔案也放入當年度的對應月份資料夾中
  - 程式透過檔名中的年月（如 `11501` vs `11401`）自動區分今年/去年
- **若找不到去年資料**，程式記錄警告訊息，去年相關參數**預設為 0**

### 3.2 來源檔案命名規則

```
{公司代號}_{民國年月4碼}_保險收入統計表.xlsx
```

範例：`12_11503_保險收入統計表.xlsx`（旺旺友聯，民國115年3月）

### 3.2 來源檔案結構（每份皆相同）

**Sheet1 `保險收入統計表`：**

| 儲存格 | 內容 | 型態 |
|--------|------|------|
| C2 | 年月（如 11503）| 數字 |
| B4 | 公司代碼（如 "12"）| 文字 |
| C4 | 公司名稱（如 "旺旺友聯"）| 文字 |
| A6:A38 | 險種代號（0100~9900，共 33 筆）| 文字/數字混合 |
| B6:B38 | 險種名稱 | 文字 |
| C6:C38 | 保險收入金額 | 整數（可正/負/零）|
| C39 | 合計 | 整數 |

**Sheet2 `歸屬`：**
- 險種代號 → 九大類別對照表
- 程式以此為分類依據

## 五、設定檔規格 (`application.yml`)

```yaml
app:
  # 路徑設定
  import-dir: ./import              # 來源資料夾根目錄
  output-dir: ./output              # 輸出資料夾

  # 年份設定（處理哪一年的資料）
  process-year: 115                 # 民國年（留空則自動偵測）

  # 險種欄位控制
  columns:
    include-overseas-reinsurance: false   # 是否在「總」表包含國外分進(9900)
    # 未來可擴充：特定險種代號的顯示/隱藏
    # hidden-codes: []

  # 公司排序
```

## 六、歸屬對照表（險種分類）— 自動擴展機制

程式啟動時讀取**來源檔案中的 `歸屬` 分頁**作為分類參考。

### 自動擴展邏輯

1. 歸屬對照為**固定規則**，程式內建預設對照表
2. 若來源檔案中的歸屬分頁有新險種代號，自動新增至對應位置
3. 輸出報表欄位動態產生
4. `config` 可控制特定險種代號的顯示/隱藏
5. 若某公司來源檔缺少歸屬分頁 → 記錄錯誤 `"公司別XX，缺少歸屬分頁"` 並使用預設對照表
6. 險種代號統一以**文字（字串）**處理（來源中 3000~9900 為數字型態，需轉換）

## 七、處理流程（ETL Pipeline）

```
1. 啟動 & 讀取設定
   └→ 讀取 application.yml

2. 掃描來源資料夾
   └→ import/{年份}/{月份}/*.xlsx
      ├→ 依檔名年月區分今年/去年資料
      └→ 自動偵測最新月份

3. 驗證來源檔案
   ├→ 檔名格式驗證（{公司代號}_{年月}_保險收入統計表.xlsx）
   ├→ 檔案結構驗證（Sheet 名稱、欄位位置、險種代號）
   ├→ 公司代碼以 Sheet1!B4 為準
   ├→ 險種代號統一轉為文字型態
   └→ 產出驗證報告（缺少哪些公司/月份、歸屬分頁缺失等）

4. 讀取 & 解析
   ├→ 解析每份來源檔的險種金額
   ├→ 解析歸屬對照表（缺失時使用預設）
   └→ 儲存至記憶體資料結構

5. 計算（全部由程式直接計算，不使用 Excel 公式）
   ├→ 單月明細（直接搬移，無資料補 0）
   ├→ 累計明細（跨月加總，僅至最新月份）
   ├→ 九大類彙總（依歸屬分類加總）
   ├→ 同期比較（今年 vs 去年，成長率 = (今年/去年)-1）
   ├→ 去年=0 或 <0 時，以 1 為分母計算，並記錄警告日誌
   └→ 公司排名

6. 輸出 Excel（.xlsx 格式）
   ├→ 產生保費統計表（6 個分頁）
   │   ├→ 寫入計算結果數值
   │   └→ 套用格式（合併儲存格、框線、粗體 — 完全復刻範本）
   ├→ 產生同期比較分析表（2 個分頁）
   │   ├→ 寫入數值（含比較增減率各列）
   │   ├→ 增減原因 E 欄留空
   │   └→ 套用格式
   └→ 儲存至 output/

7. 完成
   └→ 輸出執行摘要（處理了幾家公司、幾個月份、有無異常與警告）
```
